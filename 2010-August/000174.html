I had to interrupt a fairly large copy operation done with

   star -copy -p -sparse -C src . dst

Later, I restarted it using the same command. It starts with a 
long list of

   star: current 'foo bar' newer.

messages, as expected. I think it should be possible to resume 
the transfer in this manner (star completes the current file when 
interrupted with Ctrl-C, so there is no half-copied file in the 
way).

Problem now is that this seems to go very slowly. The "current 
... newer" messages go by at the rate of about 3 of them per 
second; at times even slower.

Is there a way to do this more efficiently?

Thank you!
