OK, what  you get from chartte is just a few "free" bits from the UFS on disk 
inode.

Extended attributes are much more generic. On Solaris this is any attitional 
file in a new name space. On Linux it is with reduced functionality and gives 
only name value pairs - still more than file flags.

> > So please give a better explanation of your problem.
>
> My problem is that star does not preserve "chattr" attributes during a

> understand that "chattr" attributes are a whole different ball-game and this
> feature is not yet implemented by star.

File flags are implemented since 2001.

> I'll just have to forget about using the immutable flags...that is if I want
> them preserved during backups and restores. Pity!

Star should support to completely restore the file flags.

be restored correctly.
