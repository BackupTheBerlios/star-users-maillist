Les Mikesell <les at futuresource.com> wrote:

> I thought the problem with direct dump-like access to a live filesystem 
>    had been known for ages: 
> http://dump.sourceforge.net/isdumpdeprecated.html

Correct, the problems are known at least since BSD in 1981 introduced the 
dump code that was hacked for Linux. It is probably known even before when 
there was a similar program for a UNIX-V7 filesystem that did restore to an 
_unmounted_ target filesystem and retained inode numbers.


> By the way - does anyone know how zfs and it's incremenatal send/receive 
> compare?

ZFS send/receive work at block level. This is very effective as the FS does not 
need to scan directory content but it does not allow to "restore" single files.

If you like to do backups that allow to restore single files, star is OK. Later 
versions will add support for HFSv4 (NTFS) style ACLs.
