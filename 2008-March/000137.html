Under the explanation of the -diff option, it reads:

   With -vv and above, the directory content is compared
   also if in -dump mode.

First, 'also if' in my understanding reads like 'even'. This would mean that 
-dump mode usually would supress that comparison and that -vv can be used to 
re-enable it. However, I believe it is more meant like that:

   With -vv and above and when in -dump mode, the directory 
   content is also compared.

But I am still not sure what that means. I tested with

   star -c -diff -vv diffopts=\!times -C tree1 . tree2

and it shows whether a file is missing in tree1 as well as the missing files 
in tree2. I could not notice any difference in the output compared to when 
using -dump as an additional switch. Maybe this combination is supposed to 
display a kind of difference I did not think of.

Thank you for a clarification.
