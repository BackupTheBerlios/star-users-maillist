Thomas Antony <thomas at antony.eu> wrote:

> Hi,
>
> I can't find enough information about the backup levels.

The man page contains a really long paragraph about backup levels, please read it.


> level 0 is a full backup. what other levels do exist and what does which 
> level backup when the argument -wtardumps is defined?

See man page, star supports 99 levels (artificial limited).

> I created a snapshot of a logical volume (mounted as /) and mounted it 
> to /mnt/snap. Then i created a full backup which finished successfully.
> After the first full backup i deleted the snapshot and recreated it and 
> did another full backup. This time star prints that it is a partial dump 
> and created an empty archive.
>
> File system /
> Working dir /mnt/snap
> Dumptype    partial

A partial dump does not start at the root dir of a filesystem.

A level 0 dump always contains the whole filesystem.
